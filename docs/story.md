# ForgeCMS 用户故事

> 本文档以 **Given-When-Then** 结构描述 v1.0 核心用户场景。每个 Story 对应一个可验收的行为，是 QA 测试用例和功能完成判据的依据。

## 基线假设

> 已与 PRD v1.0 锁定决策对齐。

- 三种用户角色：**Admin**（最高权限）、**Editor**（全内容编辑，不管用户）、**Author**（仅能编辑自己的内容）
- 两种内容类型：**Post**（博客文章，有分类/标签、可评论）、**Page**（静态页面）
- 内容状态：Draft（草稿）/ Published（已发布）/ Scheduled（定时发布）
- **多语言 v1.0 内置**：默认 zh_CN + en,每语言独立翻译
- **评论 v1.0 内置**：支持游客评论,管理员审核流
- 编辑器：Filament RichEditor(TipTap 底层),保存为 HTML;可选导出 Markdown

## 1. 认证与账户

### US-001 用户登录

```
作为：已有账号的用户
当：在登录页输入正确邮箱和密码并提交
那么：跳转到管理后台首页，右上角显示用户头像与角色
```

**边界**：连续 5 次失败锁定 15 分钟；记住我勾选后 token 有效期 30 天。

### US-002 密码重置

```
作为：忘记密码的用户
当：输入注册邮箱并点"重置密码"
那么：邮箱收到带重置链接的邮件（链接 1 小时内有效）
```

### US-003 管理员管理用户

```
作为：Admin
当：在"用户管理"页点击"添加用户"并填写邮箱、角色
那么：新用户被创建，系统发送激活邮件；我可在列表看到该用户
```

**边界**：Editor/Author **不能**访问用户管理页（403）。

## 2. 内容创建与发布

### US-010 创建草稿

```
作为：Author
当：在"文章"页点"新建"，填写标题和正文（Markdown）并保存
那么：文章以"草稿"状态保存，仅作者和 Editor/Admin 可见；前台不可访问
```

### US-011 实时预览（**已撤销**）

Phase 2 评审决定：编辑器统一为 TipTap WYSIWYG（prd.md §7），所见即所得，不存在独立预览窗口。Markdown 模式已从 v1.0 scope 移除，本 story 随之撤销。

> 若未来 v1.x 接入 Markdown 编辑模式，再重新起一条 US。

### US-012 发布文章

```
作为：Author
当：在草稿状态下点"发布"
那么：文章状态变为"已发布"，前台列表与详情页立即可见
```

### US-013 定时发布

```
作为：Editor
当：编辑文章时选择未来时间并点"定时发布"
那么：文章进入 Scheduled 状态；到时间后 Laravel 调度任务自动转为 Published
```

**边界**：定时时间必须晚于当前至少 5 分钟；过时会自动改成立即发布。

### US-014 编辑自己的文章

```
作为：Author
当：点击我创建的某篇文章
那么：进入编辑页可修改正文、标题、分类、标签、发布状态
```

### US-015 不能编辑他人文章（Author）

```
作为：Author
当：尝试访问其他作者文章的编辑页
那么：403 Forbidden；列表页默认只显示"我的文章" tab
```

### US-016 软删除与恢复

```
作为：Editor 或 Admin
当：对文章执行"删除"
那么：文章进入垃圾箱（soft deleted），30 天后永久删除；期间可恢复
```

## 3. 分类与标签

### US-020 创建分类

```
作为：Editor
当：在"分类"管理页新建一个分类并设置父分类
那么：分类被创建，形成树状结构，可挂载到文章
```

**边界**：分类层级最多 3 层；同层名称不能重复。

### US-021 给文章打标签

```
作为：Author 编辑文章时
当：在标签输入框输入新标签名并回车
那么：标签自动创建并挂到文章；已存在的标签从下拉补全
```

### US-022 按标签浏览前台

```
作为：读者
当：点击文章下方某个标签
那么：跳转到 `/tags/{slug}` 页，展示所有带该标签的已发布文章列表
```

## 4. 媒体管理

### US-030 上传图片

```
作为：Author 编辑文章时
当：拖拽图片到编辑器 或 点"插入图片"选择文件
那么：图片上传到配置的存储（本地/S3），在正文中插入 Markdown 图片引用
```

**边界**：单文件 ≤ 10MB；支持 jpg/png/webp/gif/svg；自动生成 400px / 800px 两档缩略图。

### US-031 媒体库浏览

```
作为：Editor
当：进入"媒体库"页
那么：瀑布流展示所有已上传媒体；支持按日期/类型/上传者过滤与关键词搜索
```

### US-032 删除未使用媒体

```
作为：Admin
当：在媒体库点"清理未使用"
那么：系统扫描所有 Post/Page 内容，找出未被引用的媒体；预览后确认删除
```

## 5. 前台访问

### US-040 访问首页

```
作为：读者
当：访问 `https://site.com/`
那么：看到最新 10 篇已发布文章列表，按发布时间倒序；底部分页
```

### US-041 访问文章详情

```
作为：读者
当：点击文章标题或访问 `/posts/{slug}`
那么：看到完整正文、作者、发布时间、分类、标签；meta 标签含 title/description/og:image
```

### US-042 全文搜索

```
作为：读者
当：在顶部搜索框输入关键词并回车
那么：跳转 `/search?q=xxx`，展示 Meilisearch 返回的匹配文章列表，按相关度排序
```

**边界**：只搜索已发布内容；草稿/Scheduled 不出现；搜索空查询返回空结果而非全部。

### US-043 Sitemap 与 RSS

```
作为：搜索引擎爬虫 / RSS 订阅器
当：访问 `/sitemap.xml` 或 `/feed.xml`
那么：得到符合标准的 XML，包含所有已发布 Post 和 Page
```

## 6. 管理后台体验

### US-050 仪表盘概览

```
作为：Admin 登录后
当：打开后台首页
那么：看到文章总数、本周新增、最近登录用户、待处理任务（定时发布队列）
```

### US-051 后台响应式

```
作为：移动端用户
当：用手机浏览器打开后台
那么：布局自适应，主要操作（编辑、发布）可正常完成
```

**边界**：媒体库和用户管理优先桌面端体验。

## 7. 多语言内容

### US-060 创建文章的另一种语言翻译

```
作为：Editor 编辑已有 Post 时
当：在"翻译"标签页选择语言(如 en)并填写对应标题、slug、正文
那么：翻译以独立 `post_translations` 行保存；前台访问 `/en/posts/{slug-en}` 可见；访问 `/posts/{slug-zh}` 仍显示中文版
```

**边界**：同一语言只能有一条翻译;提交重复触发应用层校验。

### US-061 前台语言切换

```
作为：读者
当：点击页面顶部语言切换器选另一语言
那么：跳转到当前内容的对应翻译版本；若无翻译则跳转到该语言首页并显示提示
```

### US-062 `hreflang` 标签自动生成

```
作为：搜索引擎
当：爬取文章详情页
那么：HTML `<head>` 包含所有该文章已发布翻译的 `<link rel="alternate" hreflang="xx">`
```

### US-063 管理后台语言切换

```
作为：Admin / Editor
当：在 Filament 后台头部切换界面语言
那么：后台 UI 文案立即切换；内容数据不受影响(每条翻译独立保存)
```

## 8. 评论系统

### US-070 游客发表评论

```
作为：未登录的读者
当：在文章底部表单填写昵称、邮箱、内容并提交
那么：评论以 `pending` 状态保存;前台提示"审核后可见",暂不显示
```

**边界**：蜜罐字段防机器人;同 IP 每分钟≤2 条;邮箱格式必须合法但不强制验证真实性。

### US-071 注册用户发表评论

```
作为：已登录用户
当：同样提交评论
那么：若站点配置"登录用户免审"则直接 `approved`;否则走审核流
```

### US-072 管理员审核评论

```
作为：Admin 或 Editor
当：进入 Filament "评论管理"页,看到 `pending` 列表
那么：可"通过" / "标记垃圾" / "删除";"通过"后评论立即出现在前台
```

### US-073 嵌套回复评论

```
作为：读者(游客或登录)
当：在某条已审核评论下点"回复"并提交
那么：新评论以 `parent_id` 指向被回复评论;前台以缩进层级显示(最多 3 层)
```

### US-074 禁用单篇评论

```
作为：Author 或 Editor 编辑文章时
当：取消"允许评论"开关并保存
那么：前台该文章评论区隐藏或显示"已关闭评论";已有评论仍保留但不接受新评论
```

### US-075 Akismet 集成(可选)

```
作为：Admin 在站点设置里填入 Akismet Key
当：有新评论提交
那么：自动调用 Akismet API,返回 spam 的直接标记 `status=spam`;非 spam 照常走审核
```

**边界**：Akismet 故障时降级为正常审核流程,不阻塞评论提交。

## 9. 非功能性验收

### US-090 页面加载性能

```
当：首次冷启动访问首页
那么：Lighthouse Performance ≥ 90，LCP < 2.5s
```

### US-091 SEO 可爬性

```
当：Googlebot 爬取任一已发布文章详情页
那么：不依赖 JavaScript 即可获取完整标题、正文、meta tags
```

### US-092 安全基线

```
当：未登录用户访问任何 `/admin/*` 路径
那么：302 跳转到登录页；POST 请求需 CSRF token，无 token 返回 419
```
