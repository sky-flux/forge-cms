# Install MVP Dependencies Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install and configure the Tier 1 + Tier 2 Composer packages (10 total) required for forge-cms v1.0 MVP, using a test-first workflow where each package group gets an integration smoke test that fails before install and passes after.

**Architecture:** Each package group is a single task = one commit. Tests are Pest feature tests in `tests/Feature/Deps/` covering "does the package's public API actually load and work in this app". Installation mutates multiple files (composer.json, config/, migrations, service providers); tests lock in that the integration is correct, not just that `composer require` succeeded.

**Tech Stack:** PHP 8.5 / Laravel 13 / Pest 4 / Postgres 18 (dev) / SQLite :memory: (tests, per phpunit.xml) / Filament 5 / Spatie suite / FrankenPHP + Octane / Horizon / Meilisearch 1.13.
